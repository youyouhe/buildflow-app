# BuildFlow Logo 使用指南

## 🎨 新 Logo 设计说明

### 设计方案：螺旋上升
**设计理念**：三个渐变方块沿虚线路径螺旋上升，象征"持续构建、不断进步"的核心理念。

### 设计元素
- **三个方块**：代表 Build（构建）的三个阶段
  - 左下蓝色方块：输入/起始
  - 中间紫色方块：处理/转换
  - 右上粉色方块：输出/完成

- **螺旋路径**：代表 Flow（流动）的过程
  - 虚线连接：技术感、工程化
  - 连接点：关键节点标识

- **成功标记**：右上角绿色勾号
  - 表示构建成功、部署完成

### 颜色体系
```
主渐变：#06B6D4 (青色) → #3B82F6 (蓝色)
次渐变：#3B82F6 (蓝色) → #8B5CF6 (紫色)
强调色：#8B5CF6 (紫色) → #EC4899 (粉色)
成功色：#10B981 (绿色)
```

## 📁 文件位置

### 主要 Logo 文件
- `public/logo.svg` - 网站公共访问 Logo（带动画）
- `assets/logo.svg` - 组件内引用 Logo（带动画）
- `public/logo-static.svg` - 静态版本（无动画，用于打印、邮件等）

### 备份文件
- `public/logo-old-backup.svg` - 原 Logo 备份
- `assets/logo-old-backup.svg` - 原 Logo 备份

## 🔧 技术规格

### SVG 属性
- 尺寸：32×32px (viewBox)
- 格式：SVG 矢量图
- 特性：支持无损缩放
- 动画：CSS 动画（opacity 呼吸效果，3秒循环）

### 兼容性
- ✅ 所有现代浏览器
- ✅ Next.js Image 组件
- ✅ 深色/浅色主题
- ✅ Retina 屏幕

## 📱 使用场景

### 1. 网站 Header
```tsx
import Logo from "@/assets/logo.svg";

<Image src={Logo} alt="BuildFlow" width={32} height={32} />
```

### 2. Favicon
文件位置：`public/logo.svg`
```html
<!-- 自动引用 -->
<link rel="icon" href="/logo.svg" />
```

### 3. 邮件/文档（静态版本）
```html
<img src="/logo-static.svg" alt="BuildFlow" width="32" height="32" />
```

### 4. 不同尺寸
Logo 支持多种尺寸，建议使用以下标准尺寸：
- **16px** - Favicon, 小图标
- **24px** - 导航栏（移动端）
- **32px** - 标准尺寸（推荐）
- **48px** - 大图标
- **64px** - 应用图标
- **128px+** - 营销页面

## 🎯 品牌规范

### 使用建议
✅ **推荐**
- 保持 Logo 周围有足够留白（至少 8px）
- 在深色背景使用（最佳效果）
- 与品牌色系搭配使用

❌ **避免**
- 不要拉伸或压缩 Logo
- 不要改变 Logo 颜色
- 不要在 Logo 上添加效果（阴影、描边等）
- 不要旋转或倾斜 Logo

### 最小尺寸
- 屏幕显示：最小 16px
- 打印输出：最小 12mm

## 💡 设计师备注

这个 Logo 的设计融合了现代设计趋势和 BuildFlow 的核心价值：
1. **可视化工作流**：通过三个方块和路径直观展示构建流程
2. **渐进增强**：颜色从冷色调到暖色调，象征项目从想法到成品的演进
3. **动态呼吸**：subtle 的透明度动画让 Logo 更有生命力
4. **技术美学**：虚线路径和几何图形体现工程师审美
